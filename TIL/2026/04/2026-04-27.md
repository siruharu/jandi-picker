## 🔍 학습한 것
- 자바를 이용해 개발하기 위해서는 JVM 을 필수로 알아야함
- 자바는 하드웨어 플랫폼이라는 족쇄를 제거하여 "한 번 작성하면 어디서든 실행된다"라는 이상을 실현한다.
- 런타임에 핫 코드(빈번하게 실행되어 전체 성능에 영향을 크게 주는 코드)를 감지, 컴파일하고 최적화하여서 자바 애플리케이션이 최상의 성능을 내도록 도와줌
- 자바 프로그래밍 언어, 자바 가상 머신, 자바 클래스 라이브러리를 묶어서 JDK 라고 한다.
  - JDK는 자바 프로그램 개발에 필요한 최소한의 환경
  - JDK는 자바 기술 시스템 전체 세대를 지칭할 때도 자주 쓰임
- 자바 SE API, 자바 가상 머신 그리고 배포 기술까지 묶어 JRE라고 한다.
  - JRE는 자바 프로그램을 실행할 수 있는 표준 환경을 제공
- https://docs.oracle.com/javase/7/docs/index.html
- 해당 스크린샷은 JDK 7 버전의 문서에 그려져있는 JDK 그림이다.
- 여기서 크게 변경되는부분은 잘없어서 해당 그림으로 대체한다.
- 자바 기술 시스템을 네 가지 주요 제품군으로 나눌 수 있음
  - 자바 카드: 스마트 카드와 같은 소형 기기 및 변조 방지 보안 칩등에서 실행되는 플랫폼
  - 자바 ME: 휴대 전화, PDA같은 모바일 기기, 예전에는 (자바5) J2ME라 불림 , 안드로이드용 어플리케이션과 아무 관련없음
  - 자바 SE: 데스크톱 어플리케이션용 자바 플랫폼, 완전한 형태의 자바 핵심 API 제공, JDK5까지는 J2SE라는 이름이었음
  - 자바 EE: 전사적 자원 관리(ERP), 경영 정보 시스템(MIS), 고객 관계 관리(CRM) 다중 계층 구조로 이루어진 기업 규모 어플리케이션용 자바 플랫폼 , JDK 10 부터는 자카르타 EE로 개명
  - 핫스팟 가상 머신의 출신일 (1999,4,27)
  - 처음에는 JDK1.2에 추가되는 형태로 제공됐다가 JDK 1.3부터 기본 자바 가상 머신으로 승격됨.
- JDK1.4
  - 성숙한 단계
  - 정규 표현식, 예외 연쇄, NIO, 로그 클래스, XML 파서, XSLT 변환기 등 다수의 신기술
- JDK 5
  - 코딩 편의성을 개선하는 큰 폭의 변화
  - 오토박싱, 제네릭스, 동적 애너테이션, 열거형, 가변 길이 매개 변수, foreach 순환문등의 문법 변화
  - 가상 머신과 API 수준에서는 자바 메모리 모델, java.util.concurrent 패키지 도입
- JDK 8
  - 모던 자바의 시작
  - 람다식 지원
  - 나스혼 자바스크립트 엔진 내장
  - 새로운 시간 및 날짜 API
  - 핫스팟에서 영구 세대 완전 제거
- JDK 11
  - ZGC 실험 버전 추가
- JVM
  - 런타임 데이터 영역 (Run-Time Data Areas) -> JVM은 프로그램 실행 중에 사용하는 다양한 런타임 데이터 영역을 정의함, 두 분류로 나뉨 -> 포인트는 왜 공유 vs 스레드별 왜이렇게 나뉘는지 이해해야함
  - PC 레지스터 (The pc Register)
  - JVM 스택 (Java Virtual Machine Stacks)
  - 힙 (Heap)
  - 메서드 영역 (Method Area)
  - 런타임 상수 풀(Run-Time Constant Pool)
  - 네이티브 메서드 스택(Native Method Stacks)
  - jvm 기술 개요
